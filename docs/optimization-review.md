## 1. 剧本创作流程

### 1.1 缺少"世界观/设定"前置环节

**现状：** 用户只能输入一个 `idea`（创意概念），系统直接生成完整剧本。

**问题：**
- 专业影视制作中，世界观设定（world building）是剧本之前的独立环节
- 用户无法预先定义时代背景、科技水平、社会规则、魔法体系等
- 系统生成的剧本缺乏一致的世界观约束，容易出现设定矛盾

**建议：**
- 在 `idea` 和 `script` 之间增加 `worldSetting` 字段（时代、地理、规则、禁忌等）
- 生成剧本时将世界观作为硬约束注入 prompt
- 后续所有环节（角色、分镜、画面）都继承世界观设定

### 1.2 缺少剧本大纲阶段

**现状：** 从 idea 一步直达完整剧本，中间没有大纲确认环节。

**问题：**
- 专业编剧流程是 概念 → 大纲 → 分场 → 完整剧本
- 一步到位意味着如果方向偏了，用户需要完全重写
- 长篇剧本（10分钟+）直接生成质量不可控

**建议：**
- 增加"大纲生成"步骤：idea → outline（3-5个关键节拍/转折点） → 用户确认 → 完整剧本
- 大纲可以包含：主线、每幕核心冲突、高潮点、结局走向
- 用户可以修改大纲后再生成，大幅减少返工

### 1.3 缺少节奏/时长控制机制

**现状：** `SCRIPT_GENERATE_SYSTEM` 中虽提到"每场景=5-15秒"，但缺乏整体时长规划。

**问题：**
- 用户想做一个2分钟的短片 vs 一个10分钟的中篇，生成的场景数量应该完全不同
- 当前 prompt 有提到 `"目标时长"` 检测，但只在生成阶段，没有在数据模型中体现
- 无法在项目级别设定目标时长

**建议：**
- 在项目/剧集层面增加 `targetDuration` 字段（秒）
- 生成剧本时根据目标时长计算场景数量范围
- 分镜阶段根据目标时长分配镜头数和每镜头时长
- UI 上展示"预计总时长"vs"目标时长"的对比

### 1.4 缺少情感曲线/节奏可视化

**现状：** 剧本编辑器是纯文本 textarea。

**问题：**
- 导演需要直观看到整个故事的情感节奏曲线（tension curve）
- 当前无法快速判断"哪里太平淡、哪里高潮堆叠太密"
- 编剧修改剧本时缺乏结构性引导

**建议：**
- 可选功能：AI 分析剧本后生成情感曲线图（每场景打分：紧张度、情感强度）
- 在分镜列表旁展示节奏线图，帮助调整 pacing

## 2. 角色设计系统

### 2.1 缺少角色关系图谱

**现状：** 角色是独立的个体，只有 `main/guest` 分类。

**问题：**
- 影视制作中角色关系（敌对、亲密、师徒、暗恋）直接影响镜头构图和场面调度
- AI 在生成两人同框画面时，不知道他们的关系，无法调整人物间距、眼神方向、肢体语言
- 对抗场景中角色应该处于对立构图，亲密关系应该靠近——当前系统没有这个信息

**建议：**
- 增加 `characterRelations` 表：characterA, characterB, relationType (对抗/亲密/陌生/敬仰/...)
- 分镜生成时注入角色关系信息，影响构图提示
- 可选：生成角色关系图谱的可视化

### 2.2 角色视觉描述缺少"表演风格"

**现状：** `CHARACTER_EXTRACT_SYSTEM` 生成了非常详细的外观描述（面部、服装、武器等），但缺少表演维度。

**问题：**
- 角色的表演风格（内敛 vs 外放、快速 vs 沉稳）影响每一帧的动作幅度
- `personality` 字段只有2-3个特征词，不足以指导帧生成和视频生成
- 不同情绪下的典型表情/动作模式没有预定义

**建议：**
- 扩展角色描述增加 `performanceStyle` 字段：
  - 动作幅度（大开大合 / 收敛内敛）
  - 标志性动作（习惯性推眼镜、抱臂、攥拳等）
  - 情绪表达模式（高兴时嘴角上扬 / 紧张时咬嘴唇 / 思考时摸下巴）
- 在 motionScript 生成时将 performanceStyle 作为参考

### 2.3 角色换装/状态变化未建模

**现状：** 每个角色只有一套外观描述和一张参考图。

**问题：**
- 真实影视中角色会换装（日常装→战斗装→礼服）
- 同一角色在不同场景可能有伤痕、脏污、雨水等状态变化
- 当前系统强制要求"所有帧中角色服装必须和参考图完全一致"

**建议：**
- 支持角色多套"造型方案"（costume set），每个 shot 可关联特定造型
- 角色状态可随剧情发展变化（受伤、变装、觉醒等）
- 参考图生成也应支持多造型

### 2.4 缺少角色身高/体型对比图

**现状：** 角色参考图是各自独立的四视图，没有放在一起的对比图。

**问题：**
- 多角色同框时，AI 不知道角色之间的相对身高和体型比例
- 两个角色描述都写了"高大"但没有相对参照
- 容易出现"小孩和大人一样高"的视觉错误

**建议：**
- 生成角色对比图（lineup sheet）：所有主要角色站在一起的身高对比参考
- 或在角色描述中明确写上 cm 级身高数据

## 3. 分镜拆解与镜头语言

### 3.1 缺少场景转换/转场标记

**现状：** `shot-split` 只生成独立的镜头列表，没有转场信息。

**问题：**
- 专业影视中，镜头之间的转场（切、叠化、淡入淡出、擦除、跳切）是叙事节奏的重要组成
- 当前所有镜头直接拼接（硬切），没有任何转场效果
- 场景转换时（从室内到室外）缺少过渡标记

**建议：**
- 在 shot 数据中增加 `transitionIn` 和 `transitionOut` 字段
- 值域：`cut` / `dissolve` / `fade_in` / `fade_out` / `wipe` / `jump_cut`
- 分镜生成时由 AI 基于叙事节奏推荐转场类型
- 视频合成时 FFmpeg 根据转场标记添加相应效果

### 3.2 镜头运动维度不足

**现状：** `cameraDirection` 只有单一选项（pan left / dolly in / orbit right 等）。

**问题：**
- 真实的镜头运动往往是复合运动（dolly in + tilt up + slow zoom）
- 当前只能选一个，限制了镜头表现力
- 手持、Steadicam、轨道、航拍等拍摄方式未区分

**建议：**
- `cameraDirection` 改为支持复合描述（自由文本），不限于固定选项
- 增加 `cameraRig` 字段：`handheld` / `tripod` / `steadicam` / `crane` / `drone` / `dolly_track`
- 拍摄方式影响画面的稳定性和质感，应该在 prompt 中体现

### 3.3 缺少景深/焦点控制

**现状：** 帧描述中有粗略的构图描述，但缺少精确的焦点信息。

**问题：**
- 焦点变化（rack focus）是非常重要的叙事手法
- 前景虚化、背景虚化、浅景深/深景深直接影响视觉效果
- AI 图像生成器可以响应景深指令，但当前 prompt 没有系统性地包含

**建议：**
- 在 shot 数据中增加 `focalPoint` 字段（焦点在哪个角色/物体上）
- 增加 `depthOfField` 字段：`shallow` / `medium` / `deep`
- 支持焦点转换指令（从A转到B）在 motionScript 中体现

### 3.4 缺少音效/配乐标记

**现状：** 只有对白，没有任何音效和配乐信息。

**问题：**
- 音效（脚步声、雨声、爆炸声）是影片情绪的重要组成
- 配乐的节奏决定剪辑节奏（所谓"跟着音乐剪"）
- 当前系统完全忽略了声音层面

**建议：**
- 在 shot 数据中增加 `soundDesign` 字段（场景环境音描述）
- 增加 `musicCue` 字段（配乐提示：紧张弦乐渐强 / 钢琴独奏 / 沉默）
- 后续可对接音效/配乐生成 AI（如 Suno、Udio）
- 视频合成时叠加音效/配乐轨道

### 3.5 连续性检查机制不足

**现状：** `shot-split` prompt 中要求"endFrame of shot N logically connects to startFrame of shot N+1"，但只是文字要求。

**问题：**
- 纯靠 prompt 约束，AI 经常违反连续性（角色位置跳跃、服装突变、光线方向不一致）
- 没有自动化的连续性检查
- 用户需要逐帧手动检查

**建议：**
- 增加连续性检查步骤：生成 N+1 帧后，用 AI 视觉模型对比 N 的 lastFrame 和 N+1 的 firstFrame
- 检查项：角色服装一致性、位置合理性、光线方向、色调匹配
- 不通过则自动重新生成并标记问题
- 可以作为可选的"高级模式"

## 4. 画面生成（关键帧）

### 4.1 缺少构图模板/参考系统

**现状：** 帧生成完全依赖 AI 自由发挥构图。

**问题：**
- 专业导演会使用经典构图参考（黄金分割、三分法、对称构图、引导线构图等）
- 同一场景的不同情绪需要不同构图策略
- AI 自由生成容易导致构图千篇一律

**建议：**
- 在分镜数据中增加 `compositionGuide` 字段
- 值域：`rule_of_thirds` / `golden_ratio` / `symmetric` / `diagonal` / `frame_within_frame` / `leading_lines`
- shot-split 时由 AI 基于叙事节奏推荐构图
- 帧生成 prompt 中加入构图约束

### 4.2 色彩管理缺少全局一致性控制

**现状：** 每个镜头的色调由 `sceneDescription` 中的描述决定，但缺少全局色彩管理。

**问题：**
- 专业影片有统一的调色方案（color grading scheme）
- 不同场景间的色彩过渡应该是连贯的
- 当前每个镜头独立描述色彩，容易出现"一场蓝调一场暖黄"的割裂感

**建议：**
- 在项目/剧集级别增加 `colorPalette` 字段（全局色彩方案）
- 定义每个情绪状态对应的色彩倾向
- 帧生成时将全局色彩方案作为硬约束注入
- 可参考 LUT（Look-Up Table）的概念做"视觉风格预设"

### 4.3 缺少"情绪板"（Mood Board）功能

**现状：** 用户无法上传参考图片来定义视觉风格。

**问题：**
- 影视预制阶段（pre-production），导演会用情绪板来沟通视觉方向
- 文字描述有时不如一张图片直观
- 用户可能想要"像《银翼杀手》那样的色调"但无法传达

**建议：**
- 增加项目级别的"情绪板"功能：用户上传参考图片+标注
- AI 分析情绪板提取视觉风格（色调、光影、构图偏好、材质质感）
- 将提取的风格注入所有帧生成 prompt

## 5. 视频生成

### 5.1 缺少动作时序精细控制

**现状：** `motionScript` 用时间分段文字描述动作（"0-2s: ... 2-4s: ..."），但视频模型只接收一段 prose。

**问题：**
- motionScript（精细时间轴）和 videoScript（30-60词 prose）之间信息量差异巨大
- 大量精细动作描述在转换为 prose 时丢失
- 视频生成模型（Seedance/Kling/Veo）无法理解分段时间轴

**建议：**
- 重新定义 motionScript 和 videoScript 的关系：
  - motionScript 作为导演参考（给人看的）
  - videoScript 是给模型的最终输入——但应该尽可能保留 motionScript 中的关键动作节拍
- 或者探索将 motionScript 转换为结构化的运动控制参数（如果模型支持）
- 考虑长镜头拆分为多段 5s 视频再拼接，以保证动作精度

### 5.2 缺少视频质量评估/自动重试

**现状：** 视频生成后直接保存，没有质量评估。

**问题：**
- AI 视频生成常见问题：肢体扭曲、人脸崩坏、物体穿模、动作不自然
- 当前用户必须手动检查每段视频，发现问题后手动重新生成
- 批量生成时，一个错误视频需要人工发现

**建议：**
- 增加可选的"视频质检"步骤：
  - 用视觉 AI 检测画面质量（人脸完整性、肢体合理性、画面清晰度）
  - 与 firstFrame/lastFrame 做视觉相似度对比（起止帧是否被忠实还原）
- 质检不通过自动重试（最多N次），标记问题供用户审核
- 质检结果可以在 UI 上用颜色标记（绿色通过/黄色警告/红色失败）

### 5.3 缺少镜头内的音效同步

**现状：** videoScript 只描述视觉动作，不包含音效时序。

**建议：**
- 在 videoScript 或 motionScript 中标注关键音效触发点
- 例如："2s: 剑鞘碰撞声 [SFX: metal_clang]"
- 后续接入 TTS + 音效生成时可以精确对齐

## 6. 视频合成与后期

### 6.1 转场效果完全缺失

**现状：** `assembleVideo` 使用 FFmpeg concat 直接硬拼接所有镜头。

**问题：**
- 所有镜头之间都是硬切（straight cut），没有任何转场
- 场景转换时显得突兀
- 缺少淡入淡出、叠化等基础转场

**建议：**
- 利用 FFmpeg filter 实现基础转场：
  - `xfade` filter 支持 fade/dissolve/wipeleft/slideright 等
  - 在每个 shot 间根据 `transitionType` 应用对应转场
- 首帧和末帧之间增加适当的转场时长（0.5-1s）

### 6.2 字幕系统过于简陋

**现状：** 字幕是整个 shot 时长内显示"角色名: 对白"，用 FFmpeg burn-in。

**问题：**
- 一个镜头内如果有多句对白，它们全部同时显示整个镜头时长
- 没有字幕样式控制（字体、大小、位置、描边、动画）
- 烧录字幕（burn-in）不可逆——应该同时提供 SRT 外挂字幕版本
- 对白字幕和旁白字幕应该有不同的视觉样式

**建议：**
- 对白时序应该在 shot 内精确分配（基于 dialogue sequence 和时长）
- 字幕样式可配置：字体、颜色、位置（底部/顶部）、描边、背景条
- 保留 SRT 文件作为外挂字幕选项
- 区分对白字幕和旁白/画外音字幕的样式
- 支持 ASS 格式以实现更丰富的样式

### 6.3 缺少片头/片尾/字幕卡

**现状：** 视频直接从第一个镜头开始，到最后一个镜头结束。

**建议：**
- 支持自动生成片头（项目标题 + 简介）
- 支持片尾（制作者信息、AI 模型信息等）
- 支持章节间的字幕卡（黑屏 + 文字过渡）
- 支持在镜头之间插入"标题卡"（如章节标题）

### 6.4 缺少配乐轨道

**现状：** 合成的视频只有视觉和字幕，没有音频。

**建议：**
- 支持用户上传背景音乐
- 支持 AI 生成配乐（接入 Suno/Udio API）
- 音乐轨道与视频合成时做基础混音
- 支持对白 TTS + 背景音乐的多轨混合

## 7. 整体工作流与用户体验

### 7.1 工作流步骤之间缺少回退机制

**现状：** 4步线性流程（剧本 → 角色 → 分镜 → 预览）。

**问题：**
- 在分镜阶段发现剧本有问题时，回去修改剧本后，之前生成的角色和分镜不会自动更新
- 用户修改了角色描述后，已生成的帧和视频没有失效标记
- 缺少"依赖链"——不知道修改了上游数据后哪些下游产物需要重新生成

**建议：**
- 建立依赖图谱：script → characters → shots → frames → videos
- 上游变更时自动标记下游产物为"过期/需更新"（不自动删除）
- UI 上用视觉提示（如灰度/黄色感叹号）标记"过期"的帧/视频
- 支持"级联重新生成"一键更新所有过期产物

### 7.2 缺少多版本对比功能

**现状：** `storyboardVersions` 存在但只是切换查看，无法并排对比。

**问题：**
- 导演需要对比不同版本的分镜，选择最优方案
- 同一镜头可能尝试不同的构图/运镜/角色调度
- 无法 A/B 对比同一镜头的不同生成结果

**建议：**
- 增加"对比模式"：左右并排展示两个版本的同一镜头
- 支持单镜头多次生成结果的历史记录和对比选择
- 让用户标记"首选"版本

### 7.3 缺少协作/审阅功能

**现状：** 单用户系统（browser fingerprint 隔离）。

**建议：（中长期）**
- 支持项目分享链接（只读 + 评论）
- 支持逐镜头批注/反馈
- 适合"导演-编剧-制片"的协作审阅场景

### 7.4 批量操作缺少进度追踪和错误恢复

**现状：** 批量生成帧/视频时，如果中间某个失败，需要手动重试。

**建议：**
- 批量操作增加进度条（X/N 完成）
- 失败的镜头自动跳过并标记，成功的保留
- 支持"只重新生成失败的"一键操作
- 增加并发控制（同时生成几个的限制，避免 API 限流）

## 8. Prompt 工程

### 8.1 多语言混合问题

**现状：** 多处 prompt 中出现中英混合：
- `buildVideoPrompt` 中 `角色形象：` 用中文，但 `Camera:` 用英文
- `buildReferenceVideoPrompt` 中 `【对白口型】` 是中文标签
- `CRITICAL LANGUAGE RULE` 要求输出语言匹配输入，但 prompt 本身是混合语言的

**问题：**
- 当用户使用英文剧本时，prompt 中的中文标签（`角色形象`、`【对白口型】`）可能干扰模型
- 不同 AI 模型对混合语言 prompt 的响应不一致

**建议：**
- 根据检测到的剧本语言，动态切换 prompt 中的标签语言
- 或统一使用英文技术标签（AI 模型普遍对英文 prompt 响应更好）
- 提供 prompt 语言偏好设置

### 8.2 Prompt 模板复用度低

**现状：** 各个 prompt builder 中有大量重复的约束文本：
- "ART STYLE" 相关约束在 `frame-generate.ts`、`character-image.ts`、`scene-frame-generate` 中都有重复
- "REFERENCE IMAGES" 的使用说明在多处重复
- "LANGUAGE RULE" 在每个 prompt 尾部重复

**建议：**
- 提取公共 prompt 片段为可复用的 "building blocks"
- 例如：`artStyleBlock()`、`referenceImageBlock()`、`languageRuleBlock()`
- 统一修改时只需改一处

### 8.3 缺少 Prompt 效果评估

**现状：** prompt 模板可以自定义编辑，但没有效果评估机制。

**建议：**
- 增加"prompt A/B 测试"功能：同一镜头用两套 prompt 生成，对比效果
- 记录每次 prompt 修改后的生成质量评分
- 建立 prompt 版本和生成质量的关联数据

## 9. 数据模型与架构

### 9.1 Shot 表字段过于膨胀

**现状：** `shots` 表有 19 个字段，包含描述、帧路径、视频路径、提示等大量信息。

**问题：**
- 单表包含了"镜头规划"（prompt, descriptions）和"生成产物"（frames, video URLs）两类数据
- 扩展困难——每增加一个生成模式就要加字段

**建议：**
- 考虑拆分为 `shots`（镜头规划数据）和 `shot_assets`（生成产物资源表）
- 或至少将多种模式的资源用 JSON 字段归组

### 9.2 缺少 "场景" 层级

**现状：** 数据模型是 Project → Episode → Shot，没有 "Scene"（场景）层级。

**问题：**
- 影视制作中 Scene（场景）和 Shot（镜头）是不同层级
- 一个场景（如"酒馆对话"）包含多个镜头（特写、全景、反打等）
- 当前 `sceneDescription` 在每个 shot 中重复，但没有统一归属
- 跨镜头的场景一致性（同一场景的光线、布景）没有数据层面保障

**建议：**
- 增加 `scenes` 表：id, episodeId, description, lighting, colorPalette, sequence
- Shot 归属 Scene：shots.sceneId → scenes.id
- 场景级别统一管理布景、光线、色调，镜头继承

### 9.3 对白时序信息不足

**现状：** `dialogues` 表只有 text, characterId, sequence，没有精确时间戳。

**问题：**
- 不知道对白在镜头内的精确开始/结束时间
- 字幕生成时只能按镜头整体时长平分
- 无法实现口型同步

**建议：**
- 增加 `startTime` 和 `endTime` 字段（相对于镜头开始的秒数）
- 或 `startRatio` 和 `endRatio`（相对于镜头时长的比例）
- 支持 AI 自动估算对白时序（根据文本长度和语速）

### 9.4 缺少"表演/动作"标注层

**现状：** 动作信息嵌入在 motionScript 和 videoScript 的文本中。

**建议：（长期）**
- 考虑增加结构化的 `actions` 层：
  - 每个 action：character, bodyPart, motion, startTime, endTime, intensity
  - 例如：{character: "李明", bodyPart: "right_hand", motion: "拔剑", startTime: 0, endTime: 2, intensity: "explosive"}
- 可以为未来更精细的动作控制（骨骼动画、MotionDiffuse 等）铺路

## 总结

AIComicBuilder 已经实现了一套完整的 AI 漫画/动画生产管线，从剧本到成片的全流程自动化。系统的 prompt 工程水平较高（特别是 shot-split 和 character-extract 的提示词非常专业），架构设计合理（任务队列、管线模式、版本管理）。

**最大的提升空间在于三个方向：**

1. **叙事层面**：增加世界观→大纲→剧本的分步创作流程，让创作过程更可控
2. **视觉一致性**：增加场景层级、全局色彩管理、连续性检查，解决"帧间不一致"这个 AI 视频最突出的问题
3. **后期合成**：转场效果、字幕时序、音效/配乐——这些是让"AI 拼接视频"变成"完整影片"的关键差距

这些优化不需要推倒重来，可以在现有架构上渐进式增加。建议按优先级分 3-4 个迭代逐步完成。
