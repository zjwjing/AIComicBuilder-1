# AI Comic Builder v0.2.0 发布：一键上传剧本，AI 自动拆集建角，从小说到动画的完整流水线

> 上传一本小说 PDF，AI 自动提取角色、智能分集、生成剧本、制作动画 —— 全程可视化，每一步都可干预。

---

## 这次更新解决了什么问题？

v0.1.x 版本中，用户需要手动创建每一集、手动编写剧本、手动触发角色解析。如果你有一部完整的小说或长篇剧本，要拆成 8 集动画短剧，光是前期准备工作就让人头疼。

**v0.2.0 的核心目标：让 AI 接管前期工作。**

你只需要上传一个文件，剩下的交给 AI。

---

## 一、剧本导入 Pipeline：四步流水线，全程可视化

这是本次最大的新功能。打开项目，点击「上传剧本」，进入全屏向导页面：

![剧本导入](images/demo/剧本上传.png)

支持 **TXT、DOCX、PDF** 三种格式，拖拽或点击上传，最大 20MB。

上传后，AI 自动执行四个步骤：

### Step 1-2：文本解析 + 角色提取（自动连跑）

AI 从全文中提取所有角色，统计出现频次，自动判断主角/配角。每个角色都会生成**电影级视觉描述**——包含体态、面部、发型、服装、武器、色彩调色板，直接可用于 AI 图像生成。

![角色提取](images/demo/剧本上传-角色解析.png)

角色卡片可以点击切换主角/配角身份，确认后进入下一步。

### Step 3：自动分集（用户确认）

AI 根据故事结构自动拆分为多集，每集生成标题、描述、关键词和详细创意构思。每集卡片上直接展示该集出场的角色：

![自动分集](images/demo/剧本上传-自动分集.png)

分集结果可以编辑标题、删除不需要的集、调整后再确认创建。

### Step 4：创建分集

一键创建所有分集和角色，自动建立角色-分集关联关系。完成后跳转到分集列表。

**全流程日志记录**，随时可以回到导入页查看历史处理过程。

## 二、项目级角色管理

之前角色分散在各集中，没有统一视图。现在新增了**项目级角色管理页面**：

![角色管理](images/demo/角色管理.png)

- 主角和配角分区展示
- 每个角色卡片支持编辑、生成四视图、删除
- 配角标注所属分集
- 角色在项目内共享，跨集复用

分集列表页面大改版：

![分集管理](images/demo/分集管理.png)

## 四、生成流水线深度优化

这次对整个生成链路做了大量优化：

### 分镜生成：分段并发，不再截断

之前 30 个场景的剧本，AI 生成分镜时 JSON 经常被截断，只出来 10 来个。现在按 SCENE 标记自动拆分（每段 ~8 个场景），**并发调用 AI**，最后合并重新编号。每段都保留完整的视觉风格和角色描述。

### 视频提示词：双帧参考

之前只传一张参考图，AI 不知道画面要怎么变化。现在**同时传入首帧和尾帧**，AI 能准确描述从起始状态到结束状态的过渡动作。

### 角色一致性：精准匹配

- Gemini 生成首尾帧时，角色参考图标注角色名
- Prompt 强制要求"图底部有角色名，服装必须和参考图完全一致"
- 不允许替换服装（比如剧本说青色常服，不能画成龙袍）

### 批量视频提示词：并发处理

从串行改为 Promise.all 并发，30 个 shot 同时生成，速度大幅提升。

### 角色隔离

所有生成阶段统一使用 `episode_characters` 关联表，只查本集角色。不会再出现"进入第 3 集却看到全部 57 个角色"的问题。

## 五、技术细节

| 改动 | 说明 |
|------|------|
| 新增 `episode_characters` 表 | 多对多关联，角色按集精确绑定 |
| 新增 `import_logs` 表 | 导入流程日志，支持历史回顾 |
| 新增 mammoth + unpdf | DOCX/PDF 文件解析 |
| 长文本分块并发 | 角色提取、分集、分镜均支持 ~10000 字分块 |
| JSON 格式强制 | `providerOptions` 设置 `response_format: json_object` |
| extractJSON 控制字符清理 | 修复 AI 返回特殊字符导致 JSON.parse 失败 |
