# Seedance 2.0 Prompt 模式总结

> 来源：https://github.com/ZeroLu/awesome-seedance

---

## 核心结构

Seedance prompt 分为两层：**全局 Header** + **逐镜头描述**。

### 全局 Header（必须）

```
【风格】电影感夜晚，雨天，高风险运动
【时长】15秒
【角色/人物】角色A（黑色大衣，湿发，红眼眶）VS 角色B（白裙，满脸泪水）
【全局要求】9:16 竖屏，人物一致性稳定，口型与字幕逐字一致
```

关键点：
- 风格标签简短，用逗号分隔（如 `IMAX 70mm 胶片，丹尼斯·维伦纽瓦风格，低饱和度`）
- **角色描述极简**：只写关键外貌+本场情绪，不超过一句话
- 全局要求写拍摄约束，不写动作

---

## 逐镜头格式

```
[00:00-00:05] 镜头 N：标题（景别类型）
[场景/动作/情感描述 — 纯散文，30-60词]
【对白口型指导】角色名："台词内容"
```

---

## 各风格模式对比

### 1. 动作/商业风格
极简散文，直接描述画面和动作，嵌入镜头运动：

```
[00-05s] 镜头 1：老将（内景/特写）
雨水拍打着赛道上赛车的挡风玻璃。老将（戴头盔）看过来，冷静专注。
仪表盘灯光反射在面罩上。
对白提示：他微微点头，低声说"出发吧。"

[05-10s] 镜头 2：挑战者（内景/特写）
切换到旁边的对手车内。年轻车手紧握方向盘，呼吸沉重，眼神充满肾上腺素。
对白提示：他对自己说"专注。"

[10-15s] 镜头 3：绿灯（远景动作）
两辆车同步加速，水花飞溅到镜头上，运动模糊将灯光变成长长的彩色线条。
```

特点：每个镜头 15-30 词，无标签，干净直接。

---

### 2. 艺术电影风格（王家卫）
加入 `场景/人物/情感表现` 标签，镜头描述更有层次：

```
[00:00-00:04] 镜头 1：透过玻璃窥视
场景: 雨水覆盖的红色公共电话亭。
人物: 穿卡其色风衣的男人紧握听筒，没有说话，只是在听。
情感表现: 透过玻璃折射，他的眼神空洞却饱含深情。雨水流过玻璃，扭曲了他的脸，像油画。
字幕/叙事感: 画面仿佛凝固，只有雨声。
```

特点：有标签但每项仍然短（1-2句），强调**情感映射**。

---

### 3. 短剧/情感爆发风格
以动作节奏驱动，对白口型指导是核心：

```
【风格】热门网剧风，极致快切节奏，高颜值滤镜，情感爆发
【时长】15秒
【人物】深情总裁男主（黑大衣，湿发，红眼眶）VS 倔强女主（白裙，满脸泪水）

[00:00-00:05] 镜头 1：快速剪辑组合
雨街。女主决然转身离去。男主冲上前拉住手腕，两人对视。
镜头组合：雨滴特写 + 两人对视 + 男主眼眶泛红 + 女主泪水滑落。
【对白口型指导】男主嘶吼："你凭什么替我做决定！"
【对白口型指导】女主哭喊："因为我爱你啊！"
```

特点：镜头动作和情绪标签混排，口型指导单独成行。

---

### 4. 视觉奇观/VFX 风格
每个镜头有**核心视觉事件**描述 + 物理细节 + 关键细节标注：

```
[00:00-00:05] 镜头 1（揭露）
场景显示繁华城市街道。镜头抬起，露出哥斯拉大小的橘猫卡在两栋摩天大楼之间。
动作: 巨猫因太胖被卡住，挥舞巨大爪子，露出可怜表情。
细节: 阳光下猫毛清晰可见，巨大肉垫压在玻璃幕墙上，使玻璃变形。
```

## 关键规律总结

| 维度 | Seedance 做法 | 我们之前的做法 |
|------|--------------|--------------|
| 角色描述 | 极简 1 句，只写本场外貌状态 | 200+ 词完整人设 |
| 每镜头长度 | 20-60 词 | 90-130 词 |
| 格式 | 散文，标签按需加，不强制 | 强制 Scene/Action/Performance/Detail |
| 摄像机 | 嵌入散文自然描述 | 单独字段 |
| 对白 | 【对白口型指导】单独成行，在镜头内 | 独立 Dialogue Cue 块 |
| 全局角色信息 | Header 一次声明，镜头不重复 | 每个 prompt 都带完整角色 |

---

## 最重要的一条

> **视频模型已经有首帧/尾帧/参考图作为图像输入，它能看到角色长什么样。**
> Prompt 应该告诉它"发生什么"，而不是"角色长什么样"。

角色的外貌一致性靠**图像输入**来保证，不靠 prompt 中的文字描述。
文字描述只需要提供情绪状态和动作方向。

## 对我们系统的启示

### 当前问题
1. `[CHARACTERS]` 块带着完整人设（200词/人），视频模型不需要这些，还会干扰
2. `videoScript` 有过多结构标签，不是自然散文
3. 摄像机指令和动作描述是分开的，视频模型更喜欢自然嵌入

### 理想的发送给视频模型的 prompt（首尾帧模式）

```
Duration: 10s. Smoothly interpolate from opening to closing frame.

陆云舟（月白长袍，玉簪束发）从棋盘上缓缓抬眼，头微侧转向斜后方，
嘴角牵出一抹含笑弧度，纱衣随晨风轻轻摆动。镜头缓慢推近。

【对白口型】陆云舟: "既然来了，何必藏头露尾？"

[Opening frame] → [Closing frame]
```

## 增加"视频 Prompt 生成"步骤的价值

### 当前流程
```
剧本 → 分镜生成(JSON) → 首/尾帧生成 → 用 JSON 里的 videoScript 组装 prompt → 视频生成
```

### 痛点
`videoScript` 是在帧生成**之前**写的，描述的是"计划中的画面"，不是"实际渲染出来的画面"。
视频模型看到的 prompt 和它实际接收的图像可能不一致。

### 增加 prompt 生成步骤后

```
剧本 → 分镜生成(JSON) → 首/尾帧生成 → [AI 看图写 prompt] → 视频生成
```

AI 看着实际渲染出来的帧，结合剧本台词，写出准确描述帧内容的 Seedance 风格 prompt。

**适合场景**：参考图模式（有 sceneRefFrame）收益最大，因为 prompt 是唯一方向指导。
**首尾帧模式**：帧本身是图像输入，模型会优先看图，prompt 只负责描述中间动作，收益相对小。
